ReInk Linux USB read-only command checklist
============================================

Run these commands on native Linux. Send back sanitized output. Do not use
sudo, unload usblp, unbind drivers, or modify udev rules.

1. Clone and build
------------------

git clone https://github.com/fes/reink-rust.git
cd reink-rust
cargo build --workspace

If the build reports missing libusb development files, report that error rather
than installing anything yet.

2. Record environment details
-----------------------------

rustc --version
cargo --version
uname -sr
pkg-config --modversion libusb-1.0

3. Identify Epson USB products and printer interfaces
-----------------------------------------------------

lsusb -d 04b8:
lsusb -t

for dev in /sys/bus/usb/devices/*; do
  [ -f "$dev/idVendor" ] || continue
  [ "$(cat "$dev/idVendor")" = "04b8" ] || continue
  echo "Device: $(basename "$dev")  VID:PID=$(cat "$dev/idVendor"):$(cat "$dev/idProduct")"
  for intf in "$dev":*; do
    [ -f "$intf/bInterfaceNumber" ] || continue
    if [ -L "$intf/driver" ]; then
      driver="$(basename "$(readlink "$intf/driver")")"
    else
      driver=none
    fi
    echo "  interface=$(cat "$intf/bInterfaceNumber") class=$(cat "$intf/bInterfaceClass") driver=$driver"
  done
done

4. Check local printer device-file candidates and permissions
-------------------------------------------------------------

cargo run -p reink-cli -- local-devices
ls -l /dev/lp* /dev/usb/lp* 2>/dev/null || true

5. Request only the standard USB Printer Class device ID
--------------------------------------------------------

For each discovered Epson device, replace the product ID and interface number:

cargo run -p reink-cli -- usb-id \
  --vendor-id 0x04b8 \
  --product-id 0xNNNN \
  --interface N

The command never enters Epson D4 mode and never writes printer state. An
error that an active kernel driver owns the interface is an expected safe stop
condition. Record the exact error, but do not detach or rebind the driver.

6. Optional capture-only D4 entry probe
---------------------------------------

Run this only after a successful standard device-ID read on an interface with
no active kernel driver:

cargo run -p reink-cli -- usb-d4-probe \
  --vendor-id 0x04b8 \
  --product-id 0xNNNN \
  --interface N

The probe sends the source-compatible Epson entry sequence and reads at most
five bounded replies. It stops before D4 Init, service opening, EEPROM reads,
writes, or counter resets. It reports only whether the known reply was
recognized, or an unexpected reply byte count; it never prints raw response
bytes.

7. Complete read-only evidence collection
-----------------------------------------

For every printer whose selected interface has no active kernel driver, retain
sanitized JSON from the following commands:

cargo run -p reink-hardware-test -- read-sequence \
  --vendor-id 0x04b8 --product-id 0xNNNN --interface N

cargo run -p reink-hardware-test -- d4-identity \
  --vendor-id 0x04b8 --product-id 0xNNNN --interface N --model MODEL

Run the complete sequence twice on the same printer. Retain both results to
check stable replies and orderly shutdown.

After a successful D4 identity result, use d4-eeprom-read only for explicitly
selected addresses: one low address, one mid-range address, one configured
waste-counter address, and one near the configured upper bound.

Also retain sanitized errors for an invalid model name and an invalid EEPROM
address. Do not induce transport, USB, or printer errors deliberately.

For a printer blocked by an active kernel driver, retain only the driver-block
error and device-file permission output. Do not detach or rebind the driver.
