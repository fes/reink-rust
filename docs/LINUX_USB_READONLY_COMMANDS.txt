ReInk Linux USB read-only command checklist
============================================

Run this workflow on native Linux for diagnostics and repair. It performs
read-only USB evidence collection only: it never writes, restores, resets,
prints, or cleans a printer.

1. Check out the repositories as siblings
-----------------------------------------

git clone https://github.com/fes/reink-rust.git
git clone https://github.com/fes/reink-results.git

The Linux host requires `cargo`, `python3`, and the libusb development package
needed to build `reink-rust`.

2. Run the complete automated evidence collection
-------------------------------------------------

cd reink-results
./run-linux-read-evidence.sh

The runner lists descriptor-only printer candidates, chooses a candidate only
when it is unambiguous, derives the bundled model range, and collects
preflight, D4 identity, selected EEPROM reads, a bounded dump, and one
out-of-range boundary observation. For more than one candidate or model hint,
select the intended printer explicitly:

./run-linux-read-evidence.sh --candidate-alias usb-1 --model L1300

3. Linux driver handoff and recovery
------------------------------------

After a printer is explicitly selected, the read-only hardware driver
automatically performs the selected interface handoff for each operation. When
a Linux kernel driver such as `usblp` owns that interface, ReInk temporarily
detaches it, claims the interface, releases it, and reattaches only the driver
it detached. No external shell session, `sudo`, manual unbind, module unload,
or udev change is required.

If detach, claim, release, or reattachment fails, the command reports clear
recovery guidance in its console output and structured report: reconnect the
printer, power-cycle it if needed, then reboot the host before retrying. This
is a diagnostic/repair maintenance risk, not a reason to conceal or suppress
the failure. The script continues the read-only evidence sequence so each
attempted operation leaves a record.

4. Retain and share evidence safely
-----------------------------------

The runner writes raw console logs, reports, and traces to
`private-evidence/<timestamp>/`, which is ignored by Git. Keep that directory
private: it can contain printer identity, EEPROM values, USB topology, and
other device-specific data. Share only manually sanitized summaries. Before
adding a protocol fixture, manually review and redact a trace, then use
`trace-to-transcript`; that command does not sanitize input.

Lower-level selected-printer operations use the same automatic Linux handoff.
Descriptor enumeration remains descriptor-only and never opens a device or
hands off a driver.
